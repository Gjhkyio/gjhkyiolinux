HOWTO: Build your own Puppy using Woof-CE

Woof-CE is the tool to build Puppies, including the one you're using right now! This page will walk you through how to build and customize a Puppy.

First, you need at least 10 gigabytes of free storage and your Puppy's devx SFS. How to get the latter is explained here.

Open a terminal, and let's get going!

Building

Step 1: Clone Woof-CE

Run the command git clone https://github.com/puppylinux-woof-CE/woof-CE.git to clone (download) Woof-CE to your computer.

Step 2: Run merge2out

Run the command ./merge2out and follow the prompts. Choose the target architecture of your Puppy, the Linux distro whose packages will be used, and which version of the distro to use.

Step 3: Build your Puppy!

Run the command cd ../woof-out_* to go to the newly created directory, and then just run the following commands in order! ./0setup ./1download ./2createpackages ./3builddistro If everything went well, you should have a brand new Puppy .iso file in the directory! Congratulations!

Customizing

Now, this is great and all, but what if you want to customize your Puppy? Well, that's what this section is about.

The files we'll be talking about here are DISTRO_SPECS, DISTRO_PKGS_SPECS, and _00build.conf.

DISTRO_SPECS

DISTRO_SPECS is the file that specifies the name of the Puppy, its version number, the compatible distro, among other things. If you've already used merge2out, the only relevant variables are DISTRO_NAME, DISTRO_VERSION, and DISTRO_FILE_PREFIX.

DISTRO_PKGS_SPECS

This is the file that specifies the packages that your Puppy includes, you can edit this to customize the packages in your Puppy! How it's formatted is explained in comments in the file. Other files also have explanatory comments.

_00build.conf

This is the file that specifies other settings for the Puppy, including the default theme, the default apps, among other things.
